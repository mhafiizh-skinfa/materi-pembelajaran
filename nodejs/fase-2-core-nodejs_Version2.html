Step 2.1 — Pengenalan Node.js

Memahami apa itu Node.js, arsitektur, dan cara kerjanya

Apa itu Node.js?

Node.js adalah runtime environment yang memungkinkan JavaScript berjalan di luar browser — di server, komputer, atau terminal. Node.js bukan bahasa pemrograman baru, melainkan platform yang menggunakan V8 Engine milik Google Chrome untuk mengeksekusi JavaScript.

📌
Ingat!

Node.js BUKAN bahasa pemrograman. Node.js adalah runtime environment. Bahasanya tetap JavaScript.

Sejarah Singkat Node.js

1

2009 — Kelahiran Node.js

Ryan Dahl menciptakan Node.js menggunakan V8 Engine milik Google Chrome. Tujuannya: membuat web server yang cepat & ringan dengan I/O non-blocking.

2

2010 — NPM Lahir

Node Package Manager (NPM) hadir, memudahkan distribusi dan penggunaan package/library JavaScript.

3

2011 — Windows Support

Node.js mulai bisa berjalan di Windows, memperluas jangkauan pengguna secara signifikan.

4

2015 — Node.js Foundation

Dibentuk Node.js Foundation untuk mengelola pengembangan secara terbuka dan terstruktur.

5

Sekarang — Dipakai Perusahaan Besar

Netflix, LinkedIn, Uber, NASA, PayPal menggunakan Node.js untuk aplikasi production mereka.

Event Loop & Non-Blocking I/O

Ini adalah konsep TERPENTING yang membedakan Node.js dari platform lain. Node.js menggunakan model Single Thread, Non-Blocking I/O.

BLOCKING (tradisional): NON-BLOCKING (Node.js): ───────────────────── ───────────────────── Req 1 → TUNGGU selesai Req 1 → kirim ke background Req 2 → TUNGGU selesai Req 2 → kirim ke background Req 3 → TUNGGU selesai Req 3 → kirim ke background Req 2 selesai → proses hasil Req 1 selesai → proses hasil Req 3 selesai → proses hasil 😫 Seperti kasir bank: 😎 Seperti restoran: 1 kasir, antrian panjang 1 pelayan ambil semua pesanan dapur proses semua sekaligus!
✅
Keunggulan Node.js

Karena non-blocking, Node.js bisa menangani ribuan koneksi secara bersamaan dengan resource yang minimal. Sangat cocok untuk aplikasi real-time dan API server.

Module Caching

Node.js meng-cache module setelah pertama kali di-require(). Artinya pemanggilan ke file yang sama akan mengembalikan objek yang sama persis.

counter.js + app.js
JS 
// ── FILE: counter.js ──
let count = 0
module.exports = {
  increment: () => ++count,
  getCount:  () => count
}

// ── FILE: app.js ──
const counter1 = require('./counter')
const counter2 = require('./counter') // ← objek SAMA dengan counter1!

counter1.increment()
counter1.increment()

console.log(counter1.getCount()) // 2
console.log(counter2.getCount()) // 2 juga! (bukan 0)
// Karena counter1 === counter2 (module cache!)

Semantic Versioning (SemVer)

Format: MAJOR . MINOR . PATCH Contoh: 4 . 18 . 2 MAJOR → perubahan BESAR, tidak backward compatible (breaking change) MINOR → fitur BARU, masih backward compatible PATCH → bug fix kecil Simbol di package.json: ^4.18.2 → boleh update MINOR & PATCH (4.18.3 ✅, 4.19.0 ✅, 5.0.0 ❌) ~4.18.2 → boleh update PATCH saja (4.18.3 ✅, 4.19.0 ❌) 4.18.2 → harus PERSIS versi ini (exact) * → versi TERBARU (sangat tidak disarankan!)

2.4.1 — path Module

Digunakan untuk memanipulasi path file dan direktori secara aman dan lintas OS (Windows/Linux/macOS).

path-module.js
JS 
const path = require('path')

const filePath = '/home/budi/documents/laporan/data.json'

// Method-method penting
console.log(path.basename(filePath))          // data.json
console.log(path.dirname(filePath))           // /home/budi/documents/laporan
console.log(path.extname(filePath))           // .json
console.log(path.basename(filePath, '.json')) // data (tanpa ekstensi)

// path.join → gabung path (otomatis sesuai OS)
const fullPath = path.join(__dirname, 'data', 'users.json')
// Windows: C:\project\data\users.json
// Linux  : /project/data/users.json

// path.resolve → absolute path dari cwd
console.log(path.resolve('data', 'users.json'))
// /home/budi/project/data/users.json

// path.parse → pecah jadi object
const parsed = path.parse(filePath)
console.log(parsed)
/*
{
  root: '/',
  dir: '/home/budi/documents/laporan',
  base: 'data.json',
  ext: '.json',
  name: 'data'
}
*/

// path.format → kebalikan parse
const formatted = path.format({
  dir : '/home/budi/documents',
  name: 'output',
  ext : '.txt'
})
console.log(formatted) // /home/budi/documents/output.txt

// __dirname vs __filename
console.log(__dirname)   // path folder file ini berada
console.log(__filename)  // path lengkap file ini
💡
Tips Penting!

Selalu gunakan path.join() untuk menggabungkan path, jangan pakai string concatenation ('/folder' + '/file'). Ini memastikan separator yang benar di setiap OS.

2.4.3 — fs Module (File System)

Digunakan untuk membaca, menulis, menghapus, dan memanipulasi file dan direktori.

Cara 1: Synchronous (Blocking)

fs-sync.js
JS 
const fs = require('fs')

// Tulis file (sync — blocking)
fs.writeFileSync('catatan.txt', 'Halo dari writeFileSync!')
console.log('File dibuat!')

// Baca file (sync)
const isi = fs.readFileSync('catatan.txt', 'utf-8')
console.log('Isi file:', isi)

// Append (tambah ke akhir)
fs.appendFileSync('catatan.txt', '\nBaris kedua!')

// Cek apakah file/folder exist
console.log('File exist:', fs.existsSync('catatan.txt')) // true

// Info file (stat)
const stat = fs.statSync('catatan.txt')
console.log('Ukuran    :', stat.size, 'bytes')
console.log('Dibuat    :', stat.birthtime)
console.log('Is file?  :', stat.isFile())
console.log('Is folder?:', stat.isDirectory())

// Rename / Pindah
fs.renameSync('catatan.txt', 'renamed.txt')

// Hapus file
fs.unlinkSync('renamed.txt')
console.log('File dihapus!')

Cara 2: Asynchronous dengan Promise (Recommended)

fs-async.js
JS 
const fs = require('fs').promises
// atau: const fs = require('fs/promises')

const operasiFile = async () => {
  try {
    // Buat folder jika belum ada
    await fs.mkdir('data-output', { recursive: true })

    // Tulis file JSON
    const dataUsers = [
      { id: 1, nama: 'Budi', email: 'budi@email.com' },
      { id: 2, nama: 'Sari', email: 'sari@email.com' },
    ]
    await fs.writeFile(
      'data-output/users.json',
      JSON.stringify(dataUsers, null, 2), // null, 2 → pretty print
      'utf-8'
    )
    console.log('✅ File users.json dibuat')

    // Baca file JSON
    const raw  = await fs.readFile('data-output/users.json', 'utf-8')
    const data = JSON.parse(raw)
    console.log('📖 Data dibaca:', data)

    // Append ke file log
    await fs.appendFile(
      'data-output/log.txt',
      `[${new Date().toISOString()}] Program dijalankan\n`
    )

    // List isi folder
    const files = await fs.readdir('data-output')
    console.log('📁 Isi folder:', files)

    // Copy file
    await fs.copyFile(
      'data-output/users.json',
      'data-output/users-backup.json'
    )
    console.log('✅ File di-backup!')

  } catch (err) {
    console.error('❌ Error:', err.message)
  }
}

⚠️
Kapan Pakai Sync vs Async?

Gunakan Sync hanya untuk inisialisasi awal (misal: baca config saat startup). Untuk operasi dalam request/response, selalu gunakan Async agar server tidak blocking!

2.4.4 — events Module

Event Emitter adalah pola desain inti di Node.js — hampir semua objek built-in menggunakannya (fs, http, streams, dll).

events-module.js
JS 
const EventEmitter = require('events')
const emitter = new EventEmitter()

// on() → daftar listener (bisa dipanggil berkali-kali)
emitter.on('halo', (nama) => {
  console.log(`Halo, ${nama}!`)
})

// once() → listener hanya jalan SEKALI
emitter.once('login', (user) => {
  console.log(`${user} berhasil login (hanya sekali)`)
})

// emit() → trigger event
emitter.emit('halo', 'Budi')   // Halo, Budi!
emitter.emit('halo', 'Sari')   // Halo, Sari!
emitter.emit('login', 'Admin') // Admin berhasil login
emitter.emit('login', 'Admin') // tidak jalan (once)

// off() → hapus listener
const handler = (msg) => console.log('Pesan:', msg)
emitter.on('pesan', handler)
emitter.emit('pesan', 'pertama')   // jalan
emitter.off('pesan', handler)
emitter.emit('pesan', 'kedua')     // tidak jalan

// listenerCount
console.log('Listeners:', emitter.listenerCount('halo')) // 1

Class Mewarisi EventEmitter

bank-account.js
JS 
const EventEmitter = require('events')

class BankAccount extends EventEmitter {
  constructor(pemilik, saldo = 0) {
    super()
    this.pemilik = pemilik
    this.saldo   = saldo
    this.riwayat = []
  }

  setor(jumlah) {
    if (jumlah <= 0) {
      this.emit('error-transaksi', 'Jumlah setor harus lebih dari 0')
      return
    }
    this.saldo += jumlah
    const trx = { tipe: 'setor', jumlah, saldo: this.saldo }
    this.riwayat.push(trx)
    this.emit('setor', trx)
  }

  tarik(jumlah) {
    if (jumlah > this.saldo) {
      this.emit('error-transaksi', 'Saldo tidak mencukupi!')
      return
    }
    this.saldo -= jumlah
    const trx = { tipe: 'tarik', jumlah, saldo: this.saldo }
    this.riwayat.push(trx)
    this.emit('tarik', trx)
  }
}

// Penggunaan
const akun = new BankAccount('Budi', 500000)

akun.on('setor', (t) =>
  console.log(`✅ Setor Rp${t.jumlah.toLocaleString()} | Saldo: Rp${t.saldo.toLocaleString()}`)
)
akun.on('tarik', (t) =>
  console.log(`💸 Tarik Rp${t.jumlah.toLocaleString()} | Saldo: Rp${t.saldo.toLocaleString()}`)
)
akun.on('error-transaksi', (msg) =>
  console.log(`❌ Error: ${msg}`)
)

const http = require('http')

// Simulasi database (array)
let produk = [
  { id: 1, nama: 'Laptop',   harga: 8000000, stok: 10 },
  { id: 2, nama: 'Mouse',    harga: 150000,  stok: 50 },
  { id: 3, nama: 'Keyboard', harga: 350000,  stok: 30 },
]

// Helper: baca body request
const parseBody = (req) => new Promise((resolve, reject) => {
  let body = ''
  req.on('data',  chunk => body += chunk.toString())
  req.on('end',   ()    => {
    try { resolve(JSON.parse(body || '{}')) }
    catch (e) { resolve({}) }
  })
  req.on('error', reject)
})

// Helper: kirim JSON response
const sendJSON = (res, statusCode, data) => {
  res.writeHead(statusCode, { 'Content-Type': 'application/json' })
  res.end(JSON.stringify(data, null, 2))
}

const server = http.createServer(async (req, res) => {
  const { url, method } = req
  const pathname = url.split('?')[0]

  console.log(`[${method}] ${url}`)

  // GET /produk → semua produk
  if (pathname === '/produk' && method === 'GET') {
    return sendJSON(res, 200, { total: produk.length, data: produk })
  }

  // GET /produk/:id
  const matchGet = pathname.match(/^\/produk\/(\d+)$/)
  if (matchGet && method === 'GET') {
    const item = produk.find(p => p.id === parseInt(matchGet[1]))
    if (!item) return sendJSON(res, 404, { pesan: 'Produk tidak ditemukan' })
    return sendJSON(res, 200, { data: item })
  }

  // POST /produk
  if (pathname === '/produk' && method === 'POST') {
    const body = await parseBody(req)
    if (!body.nama || !body.harga)
      return sendJSON(res, 400, { pesan: 'nama dan harga wajib diisi' })
    const baru = { id: produk.length + 1, ...body, stok: body.stok || 0 }
    produk.push(baru)
    return sendJSON(res, 201, { pesan: 'Produk ditambah', data: baru })
  }

  // DELETE /produk/:id
  const matchDel = pathname.match(/^\/produk\/(\d+)$/)
  if (matchDel && method === 'DELETE') {
    const id     = parseInt(matchDel[1])
    const before = produk.length
    produk       = produk.filter(p => p.id !== id)
    if (produk.length === before)
      return sendJSON(res, 404, { pesan: 'Produk tidak ditemukan' })
    return sendJSON(res, 200, { pesan: 'Produk dihapus' })
  }

  // 404
  sendJSON(res, 404, { pesan: `Route ${method} ${url} tidak ditemukan` })
})

server.listen(3000, () => {
  console.log('🚀 Server jalan di http://localhost:3000')
  console.log('Endpoint: GET/POST /produk | GET/DELETE /produk/:id')
})

// ── Base Error ──
class AppError extends Error {
  constructor(message, code, statusCode = 500) {
    super(message)
    this.name       = this.constructor.name
    this.code       = code
    this.statusCode = statusCode
    Error.captureStackTrace(this, this.constructor)
  }
}

// ── Turunan Spesifik ──
class ValidationError extends AppError {
  constructor(message, field) {
    super(message, 'VALIDATION_ERROR', 400)
    this.field = field
  }
}

class NotFoundError extends AppError {
  constructor(resource, id) {
    super(`${resource} dengan ID "${id}" tidak ditemukan`, 'NOT_FOUND', 404)
    this.resource = resource
    this.id       = id
  }
}

class AuthError extends AppError {
  constructor(msg = 'Tidak terautentikasi') {
    super(msg, 'UNAUTHORIZED', 401)
  }
}

// ── Error Handler Terpusat ──
const handleError = (err) => {
  if (err instanceof ValidationError) {
    console.log(`⚠️  Validasi gagal di "${err.field}": ${err.message}`)
  } else if (err instanceof NotFoundError) {
    console.log(`🔍 ${err.message}`)
  } else if (err instanceof AuthError) {
    console.log(`🔐 Auth: ${err.message}`)
  } else if (err instanceof AppError) {
    console.log(`🚨 App Error [${err.code}]: ${err.message}`)
  } else {
    console.log(`💀 Unknown: ${err.message}`)
    console.log(err.stack)
  }
}

// ── Penggunaan ──
const registrasi = async (data) => {
  if (!data.nama)

  if (!data.email?.includes('@'))
    throw new ValidationError('Format email tidak valid', 'email')
  if (data.password.length < 8)
    throw new ValidationError('Password minimal 8 karakter', 'password')

  return { id: Date.now(), ...data }
}

// ── Logic ──
const hitungRata = (nilai) => {
  if (!nilai.length) return 0
  return nilai.reduce((s, n) => s + n.nilai, 0) / nilai.length
}

const getGrade = (r) => r >= 90 ? 'A' : r >= 80 ? 'B' : r >= 70 ? 'C' : r >= 60 ? 'D' : 'E'

const tambahSiswa = (nama, kelas) => {
  const siswa = load()
  if (siswa.find(s => s.nama.toLowerCase() === nama.toLowerCase()))
    throw new SiswaError(`Siswa "${nama}" sudah ada!`)
  const baru = { id: Date.now(), nama, kelas, nilai: [] }
  siswa.push(baru)
  save(siswa)
  events.emit('siswa:tambah', baru)
}

const tambahNilai = (nama, mapel, nilai) => {
  if (nilai < 0 || nilai > 100) throw new SiswaError('Nilai harus 0–100')

  if (idx === -1) throw new SiswaError(`Siswa "${nama}" tidak ditemukan`)
  siswa[idx].nilai.push({ mapel, nilai, tanggal: new Date().toLocaleDateString('id-ID') })
  save(siswa)
  events.emit('nilai:tambah', siswa[idx])
}
